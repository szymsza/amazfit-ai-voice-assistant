Amazfit Voice Assistant
=======================

Voice AI assistant for Amazfit Balance running on Zepp OS.
Ask questions by voice, get spoken answers from Claude, ChatGPT, or Groq.

Architecture
------------
Monorepo with two components:

  app/    - Zepp OS mini-app (JavaScript, runs on watch + phone Side Service)
  server/ - Backend API (single endpoint: receives audio, returns audio)

How it works:

  1. Watch: user presses button, records voice (OPUS via Recorder API)
  2. Watch -> Phone: audio sent over BLE to Side Service (runs inside Zepp companion app)
  3. Phone -> Server: Side Service makes single fetch() POST with audio bytes
  4. Server pipeline: STT (Groq Whisper) -> LLM (Claude/ChatGPT/Groq) -> TTS (Groq Orpheus + ffmpeg WAV->OPUS)
  5. Server -> Phone: returns audio response (OPUS)
  6. Phone -> Watch: audio transferred back over BLE
  7. Watch: plays response via Player API

No separate Android app needed - Side Service runs inside the Zepp companion app.

Key APIs used (watch side):
  - Recorder API (OPUS) for voice capture
  - Player API (OPUS) for audio playback
  - Messaging/TransferFile API for watch <-> phone BLE communication
  - Fetch API (Side Service) for HTTP requests to our server

Groq API Details
----------------
All Groq endpoints use: https://api.groq.com/openai/v1/
Auth header: Authorization: Bearer $GROQ_API_KEY

STT (Speech-to-Text):
  Endpoint: POST /audio/transcriptions
  Model: whisper-large-v3
  Input: multipart form with audio file + model field
  Tested roundtrip: ~0.44s

TTS (Text-to-Speech):
  Endpoint: POST /audio/speech
  Model: canopylabs/orpheus-v1-english
  Voices: austin, troy, hannah (and others)
  Output format: WAV only (16-bit PCM mono 24kHz)
  Requires server-side ffmpeg conversion: WAV -> OPUS (~260ms, 211KB -> 12KB)
  Supports vocal directions in input text: [cheerful], etc.
  Tested roundtrip: ~0.87s + ~0.26s conversion = ~1.13s total

LLM (Chat Completions):
  Endpoint: POST /chat/completions
  Default model: moonshotai/kimi-k2-instruct (0.42s, knowledge cutoff ~June 2025)
  Other models: llama-3.3-70b-versatile (0.32s), qwen/qwen3-32b, groq/compound (web search, ~10s)
  OpenAI-compatible request format
