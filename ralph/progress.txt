## Codebase Patterns
- TypeScript strict mode throughout — no `any`, no `.js` source files
- `Page.Option` only accepts `state`, `onInit`, `build`, `onDestroy` — use module-level variables for page state/widgets
- `CLICK_AREA` widget missing from `@zeppos/device-types`; access via typed intersection: `hmUI.widget as typeof hmUI.widget & { readonly CLICK_AREA: number }`
- `setProperty(prop.COLOR, number)` updates CIRCLE colour; `prop.MORE` only accepts `HmUIPropertyValue` (string|number|boolean|undefined), not objects
- `@zos/media` (Recorder/Player) not in device-types — declared in `app/global.d.ts`
- zosLoader module in `global.d.ts` uses `export { ... } from './page/gt/home/index.page.r.layout'` so layout types stay in sync automatically
- App name: "AI Voice Assistant"
- Server `/api/ask` uses `multipart/form-data` (multer memoryStorage): `audio` field = binary OPUS, `config` field = JSON string with all parameters. All previous `X-*` headers are gone.
- Server typecheck: `cd server && npm run typecheck`
- BLE watch↔side messaging uses `@zeppos/zml/base-app` (device) + `@zeppos/zml/base-side` (phone). `BaseApp` sets up `globalData.messaging` and adds `request()` to app options; call from page via `getApp()._options.request as (data: ArrayBuffer) => Promise<Uint8Array>`
- `Buffer`, `fetch`, `settings` globals are available in side service but absent from `@zeppos/device-types`; declared in `app/global.d.ts`
- `BaseSideService.onRequest(req, res)`: `req` is `ArrayBuffer` (binary content type); `res(null, Buffer.from(ab))` sends response back — must pass a Buffer (not raw ArrayBuffer) so the zml protocol's `.copy()` call works
- `@zos/fs` paths are relative to the app `/data` dir; `data://foo.opus` → path `'foo.opus'` in `openSync`/`statSync`/`writeSync`

- LLM abstraction: `registerProvider(name, fn)` + `callLLM(options)` pattern in `server/src/llm.ts`; providers register themselves, `callLLM` dispatches and caps history
- LLM providers live in `server/src/providers/`; each file calls `registerProvider` on import; `index.ts` side-effect-imports them (`import './providers/groq.js'`)
